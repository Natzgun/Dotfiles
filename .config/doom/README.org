|----------+----------+----------------------------------------|

** Deteccion automatica por proyecto

La indentacion se ajusta automaticamente segun el proyecto:

1. *editorconfig* (modulo activo en =init.el=): Lee =.editorconfig= del proyecto.
2. *dtrt-indent*: Si no hay =.editorconfig=, analiza el archivo y adivina (ej. si el proyecto usa 4 espacios, ajusta a 4).

** Snippets (Yasnippet)

=yas-indent-line= configurado como ='fixed= en modos C/C++.
Esto evita que los snippets re-indenten con =c-basic-offset=.

** Formato (format-on-save)

El formato automatico al guardar esta *desactivado* globalmente (=format= sin =+onsave= en init.el).
Ademas =lsp-before-save-edits= esta en =nil= para evitar que LSP formatee al guardar.

Para formatear manualmente:

| Tecla         | Accion                     |
|---------------+----------------------------|
| =SPC c f=     | Formatear buffer completo  |
| =SPC c F=     | Formatear region           |

Para C/C++ usa =.clang-format=. Para GLSL usa =~/.glsl_analyzer.json=.

* Emacs 30 + Tree-sitter

Emacs 30 auto-promueve =c-mode= a =c-ts-mode= y =c++-mode= a =c++-ts-mode= cuando los grammars de tree-sitter estan instalados. Doom no engancha =font-lock= ni =lsp!= correctamente en estos modos.

Fix: =my/fix-ts-modes= en =find-file-hook= que:
- Activa =font-lock-mode= y =lsp!= en C/C++ ts modes
- Sincroniza =c-ts-mode-indent-offset= con =c-basic-offset=
- Revierte =glsl-ts-mode= a =glsl-mode= clasico (fontificado roto)

*Importante:* Los keybindings deben estar en ambos mapas (=c++-mode-map= y =c++-ts-mode-map=).

** CMake: compile_commands.json

=my/find-compile-commands-dir= busca =compile_commands.json= en:
=.= > =build/= > =cmake-build-debug/= > =cmake-build-release/= > =out/= > =builddir/=

Pasa =--compile-commands-dir= a clangd automaticamente.

** CMake: Compilar y Ejecutar

=my/set-cmake-compile-command= detecta =CMakeLists.txt= buscando hacia arriba con =locate-dominating-file=.
Usa rutas absolutas para que funcione desde cualquier subdirectorio.

=my/cmake-find-executable= busca el binario en =build/= y =cmake-build-*/=, ignorando archivos internos de CMake.

=my/cmake-run= ejecuta el binario en un vterm popup.

** Copiar assets al build (CMakeLists.txt)

Para que rutas relativas como ="assets/texture.png"= funcionen sin importar desde donde se ejecute:

#+begin_src cmake
add_custom_command(TARGET MiApp POST_BUILD
    COMMAND ${CMAKE_COMMAND} -E copy_directory
    ${CMAKE_SOURCE_DIR}/assets $<TARGET_FILE_DIR:MiApp>/assets)
#+end_src

* MCP (Model Context Protocol)

Permite a Claude Code interactuar directamente con Emacs via =eval-elisp= y =get-diagnostics=.

Para iniciar: =M-x mcp-server-start=

Para permitir funciones avanzadas (necesario para debugging con Claude):

#+begin_src emacs-lisp
M-: (setq mcp-server-security-allowed-dangerous-functions
      '(with-current-buffer find-file require compile revert-buffer kill-buffer))
#+end_src
