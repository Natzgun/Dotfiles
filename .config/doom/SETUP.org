** Deteccion automatica por proyecto

La indentacion se ajusta automaticamente segun el proyecto:

1. *editorconfig* (modulo activo en =init.el=): Lee =.editorconfig= del proyecto.
2. *dtrt-indent*: Si no hay =.editorconfig=, analiza el archivo y adivina (ej. si el proyecto usa 4 espacios, ajusta a 4).

** Snippets (Yasnippet)

=yas-indent-line= esta configurado como ='fixed= en modos C/C++.
Esto evita que los snippets re-indenten con =c-basic-offset= y en su lugar respetan la indentacion del cursor actual.

*Problema resuelto:* Antes, si tenias =c-basic-offset= en 2, los snippets insertaban con 4 espacios porque yasnippet usaba el modo =auto= que re-indenta.

** Emacs 30 + Tree-sitter

Emacs 30 auto-promueve =c-mode= a =c-ts-mode= y =c++-mode= a =c++-ts-mode= cuando los grammars de tree-sitter estan instalados. Doom no engancha =font-lock= ni =lsp!= correctamente en estos modos. Se arregla con =find-file-hook= via la funcion =my/fix-ts-modes= que:

- Activa =font-lock-mode= en modos =*-ts-mode=
- Activa =lsp!= para C/C++
- Sincroniza =c-ts-mode-indent-offset= con =c-basic-offset=
- Revierte =glsl-ts-mode= a =glsl-mode= clasico (fontificado roto en ts)

*** Argumentos de clangd

#+begin_example
-j=4                          ; 4 hilos
--background-index            ; indexar en segundo plano
--clang-tidy                  ; activar diagnosticos de clang-tidy
--completion-style=detailed   ; autocompletado detallado
--header-insertion=never      ; no insertar headers automaticamente
--header-insertion-decorators=0
#+end_example

** CMake: compile_commands.json

Clangd necesita =compile_commands.json= para entender tu proyecto. La funcion =my/find-compile-commands-dir= lo busca automaticamente en:

1. =.= (raiz del proyecto, compatible con cmake-tools de Neovim)
2. =build/=
3. =cmake-build-debug/= / =cmake-build-release/=
4. =out/= / =out/build/=
5. =builddir/=

Y pasa =--compile-commands-dir= a clangd automaticamente.

** CMake: Compilar

=my/set-cmake-compile-command= detecta =CMakeLists.txt= buscando hacia arriba desde el archivo actual (=locate-dominating-file=). Asi funciona aunque estes en =src/main.cpp= y el =CMakeLists.txt= este en la raiz.

El comando generado:

#+begin_example
cmake -DCMAKE_EXPORT_COMPILE_COMMANDS=ON -B <root>/build -S <root> && cmake --build <root>/build
#+end_example

** CMake: Ejecutar

=my/cmake-find-executable= busca el binario compilado en:
- =build/src/<nombre-proyecto>=
- =build/<nombre-proyecto>=
- =cmake-build-debug/src/<nombre-proyecto>=
- =cmake-build-debug/<nombre-proyecto>=

** Formato

El formato automatico al guardar esta *desactivado* para C/C++.
Para formatear manualmente usa =SPC c f=. Si tienes un =.clang-format= en la raiz del proyecto, lo usara.

Crear un =.clang-format= base:

#+begin_src shell
clang-format -style=llvm -dump-config > .clang-format
#+end_src

* MCP (Model Context Protocol)

#+begin_src emacs-lisp
(use-package! mcp-server
  :config
  (setq mcp-server-emacs-tools-enabled 'all)
  (mcp-server-start))
#+end_src

Permite a Claude Code interactuar directamente con Emacs via =eval-elisp= y =get-diagnostics=.

Para reconectar despues de reiniciar Emacs: =M-x mcp-server-start=.

** Seguridad MCP

Para permitir funciones peligrosas (necesario para debugging remoto):

#+begin_src emacs-lisp
(setq mcp-server-security-allowed-dangerous-functions
      '(with-current-buffer find-file require compile revert-buffer kill-buffer))
#+end_src
